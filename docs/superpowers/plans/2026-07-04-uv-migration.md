# Migrate Raw Requirements Files to uv

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace three raw `requirements.txt` files under `test/docker/python_requirements/` with proper uv projects (pyproject.toml + uv.lock) per environment, enabling reliable dependency upgrades via `uv lock --upgrade`.

**Architecture:** One uv project per existing test environment (robot, appdaemon, homeassistant). Each project gets its own pyproject.toml declaring direct dependencies and a committed uv.lock for reproducible installs. Dockerfile and setup_virtualenv.sh switch from `pip`+`virtualenv` to `uv sync` (robot, appdaemon) and `uv export`+`uv pip install --no-deps` (homeassistant, preserving HASS's --no-deps quirk). The three projects live under `test/dependencies/<env>/`. The old `test/docker/python_requirements/` directory is removed.

**Tech Stack:** uv (latest), Python 3.11 (homeassistant), Python 3.12 (appdaemon, robot), Home Assistant 2023.5.4, AppDaemon 4.5.13, Robot Framework 7.4.2.

## Context

Current state:
- `test/docker/python_requirements/robot.txt` — 7 unpinned deps. Used by CI Docker (system pip) and `setup_virtualenv.sh` (local venv).
- `test/docker/python_requirements/appdaemon.txt` — 46 pinned deps. Used by CI Docker and `setup_virtualenv.sh`.
- `test/docker/python_requirements/homeassistant.txt` — 45 pinned deps. Installed with `--no-deps` (HASS bundles/conflicts with many transitives). Used by CI Docker and `setup_virtualenv.sh`.
- `test/docker/Dockerfile` — 11-line single-stage Python 3.11 image. No uv.
- `test/setup_virtualenv.sh` — bash script, calls `python3.11`/`python3.12` + `virtualenv` + `pip install -r ...`.

Target state:
- `test/dependencies/robot/{pyproject.toml,uv.lock}` — direct deps listed; uv resolves transitives.
- `test/dependencies/appdaemon/{pyproject.toml,uv.lock}` — pinned direct deps; uv resolves transitives.
- `test/dependencies/homeassistant/{pyproject.toml,uv.lock}` — direct dep `homeassistant==2023.5.4`; uv.lock pins transitives. Install uses `uv export | uv pip install --no-deps -` to keep HASS's --no-deps behavior.
- Dockerfile uses `uv` (installed via official image) for all three envs.
- `setup_virtualenv.sh` uses `uv venv` + `uv sync` / `uv export | uv pip install --no-deps -`.
- Upgrade workflow documented in AGENTS.md: edit pyproject.toml `dependencies`, run `uv lock --upgrade`, commit uv.lock.

Constraints:
- HASS 2023.5.4 requires Python 3.11 exactly. Cannot move to 3.12 without changing HASS version.
- AppDaemon 4.5.13 requires Python 3.10-3.13. Stay on 3.12.
- Robot Framework 7.x requires Python 3.8+. Use 3.12 for consistency with appdaemon.
- `test/.venv` is used to run robot tests via `test/.venv/bin/python -m robot ...`. Local venv must exist after `setup_virtualenv.sh robot`.

### Task 4: Create homeassistant requirements.txt

**Files:**
- Create: `test/dependencies/homeassistant/requirements.txt`

**Note (deviation from original plan):** The HASS env is installed with `--no-deps` because HASS 2023.5.4 transitively pins versions (e.g. `python-slugify==4.0.1`, `lru-dict==1.1.8`) that conflict with the versions in the existing `homeassistant.txt` (e.g. `python-slugify==6.1.2`, `lru-dict==1.2.0`). The original `pip install --no-deps` masked these conflicts. uv's lockfile generation does transitive resolution and refuses to produce a lockfile for an inconsistent set.

We can't model HASS as a normal uv project without either (a) aligning our pins to HA's pins (changes what's installed — out of scope) or (b) using `[tool.uv] override-dependencies` to force our pins (masks real conflicts). The cleanest path: HASS uses a flat `requirements.txt` (effectively its own lockfile) and is installed with `uv pip install --no-deps -r requirements.txt`. The file is committed and updated by hand when upgrading. This matches the original `pip install --no-deps -r homeassistant.txt` behavior exactly, just with `uv pip` as the installer.

### Task 5: Replace setup_virtualenv.sh with uv-based version

**Files:**
- Modify: `test/setup_virtualenv.sh`

- [ ] **Step 1: Replace file content**

Replace `test/setup_virtualenv.sh` with:

```bash
#!/bin/bash

set -e
script_dir=$(readlink -e "$(dirname "$0")")
cd "$script_dir"

if ! command -v uv >/dev/null 2>&1; then
    echo "uv is not installed. Install from https://docs.astral.sh/uv/getting-started/installation/" >&2
    exit 1
fi

function setup_sync() {
    local python="$1"
    local venv_path="$2"
    local deps_dir="$3"
    rm -rf "$venv_path"
    uv venv --python "$python" "$venv_path"
    (
        cd "$deps_dir"
        VIRTUAL_ENV="$script_dir/$venv_path" uv sync --no-install-project --active
    )
}

function setup_hass() {
    local venv_path="AppDaemonIntegrationTest/.hass"
    rm -rf "$venv_path"
    uv venv --python python3.11 "$venv_path"
    VIRTUAL_ENV="$script_dir/$venv_path" \
        uv pip install --no-deps -r "$script_dir/dependencies/homeassistant/requirements.txt"
}

robot=
hass=
appdaemon=

if [[ "$1" == all ]]; then
    robot=1
    hass=1
    appdaemon=1
else
    for env in "$@"; do
        case "$env" in
            robot)
                robot=1
                ;;
            hass)
                hass=1
                ;;
            appdaemon)
                appdaemon=1
                ;;
        esac
    done
fi

if [[ -n "$robot" ]]; then
    setup_sync python3.12 .venv dependencies/robot
fi

if [[ -n "$hass" ]]; then
    setup_hass
fi

if [[ -n "$appdaemon" ]]; then
    setup_sync python3.12 AppDaemonIntegrationTest/.appdaemon dependencies/appdaemon
fi
```

```markdown
#### Setting up virtual environment

To set up virtual environments to run the tests, use this command:

\`\`\`sh
./test/setup_virtualenv.sh robot|hass|appdaemon|all
\`\`\`

This command removes each virtualenv if it exists, then reinstalls it using `uv`.

Requires `uv` to be installed: <https://docs.astral.sh/uv/getting-started/installation/>.

#### Upgrading dependencies

Each test environment is a uv project under `test/dependencies/<env>/` with a `pyproject.toml` (direct dependencies) and `uv.lock` (pinned transitives).

To upgrade all dependencies for one environment to their latest compatible versions:

\`\`\`sh
cd test/dependencies/<env>
uv lock --upgrade
\`\`\`

To upgrade a single package:

\`\`\`sh
cd test/dependencies/<env>
uv lock --upgrade-package <name>
\`\`\`

Then commit the updated `uv.lock` (and `pyproject.toml` if it changed). To add or remove a dependency, edit `dependencies` in `pyproject.toml` and run `uv lock`.
```
